PACKAGE / MEMBERSHIP / RECOGNITION MODEL RECONCILIATION

Pause before any Sale V2b deployment or package UI redesign.

We need to make sure the product model is correct before redeploying.

Founder clarification

Membership is not a fixed package tier.

A seat is created by the first qualifying purchase.

The chapter/era defines the minimum entry amount.

Purchase packages are suggested purchase amounts, not separate membership types.

Recognition/rank is based on cumulative purchases.

Definitions to test against current code

1. Seat

* one per wallet
* created by first qualifying buy
* not repeated on later buys

2. Chapter / Era minimum

* current minimum amount required for any buy
* Genesis minimum = $5
* future chapters may increase minimum
* if current minimum is $100, then $5/$25 packages should not be selectable

3. Purchase amount

* how much USDC the user buys now
* may be $5, $25, $100, $500, $2,500, $10,000, etc.
* must be >= current chapter minimum
* must be <= anti-abuse cap

4. Recognition / rank

* derived from cumulative USDC/SYN purchases
* not the same as one purchase package
* buying $25 twice should count as $50 cumulative recognition
* buying $5 first then $25 later should count as $30 cumulative recognition
* buying $10,000 first should create one seat and high recognition immediately

5. Package labels
   Current names like Citizen / Operator / Vanguard may be confusing if they imply permanent membership tiers.

Consider neutral purchase labels:

* Starter
* Builder
* Supporter
* Backer
* Major Supporter
* Cornerstone

Or another better naming system.

But recognition/rank can remain separate.

* Chapter = minimum + rate + visual mark

Improve this if your codebase suggests a better model.

* $5 and $25 unavailable

E. How should first purchase work?

Confirm intended behavior:

* If user buys $25 first in Genesis, they get one seat + $25 cumulative recognition.
* If user buys $10,000 first, they get one seat + $10,000 cumulative recognition.
* If user later buys again, no second seat, only cumulative recognition increases.
