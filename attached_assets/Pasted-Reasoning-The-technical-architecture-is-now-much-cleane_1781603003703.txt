Goal

Map The Syndicate’s human experience journey and compare it to what already exists.

The protocol should not only answer:

“What can I click?”

It should answer:

* Who am I here?
* What stage am I in?
* What did I just do?
* What changed because of it?
* What should I do next?
* Why should I come back?
* What can I share?
* What mark did I leave?

A. Human journey model

Map the intended human journey:

1. Observer
   Someone discovering the protocol.

2. Explorer
   Someone browsing proof, activity, artifacts, members, token, liquidity, Chronicle/Record.

3. Connected visitor
   Wallet connected, identity known, not yet a member.

4. Member
   Bought through Sale V2, received SYN, appears in Holder Index.

5. Collector
   Owns Archive1155 artifact(s).

6. Builder
   Recognized through protocol participation/growth, but without financial-return framing.

7. Protocol Figure / Witness
   High-recognition member whose activity, artifacts, referrals, Chronicle moments, or records make them visible in protocol history.

For each stage, define:

* meaning
* how the code can detect it today
* whether it is LIVE / PARTIAL / PREVIEW / PENDING / NOT BUILT
* what page/surface currently expresses it
* what the user should see
* what the next best action should be
* what should be shareable
* what must not be implied

C. Progression vs action

Separate clearly:

Protocol Action:
A thing the user can do now.

Examples:

* buy membership
* mint artifact
* verify
* trade
* add liquidity

Progression:
The user’s evolving story.

Examples:

* became member
* joined Chapter I
* reached a rank
* collected artifact
* witnessed milestone
* appeared in record
* invited members later
* returned after protocol changed

Explain where the current code supports progression and where it only supports action.
