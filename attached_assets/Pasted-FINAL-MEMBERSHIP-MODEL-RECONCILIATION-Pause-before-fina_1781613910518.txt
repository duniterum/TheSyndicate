Current founder intent

Please use this as the reference model and verify whether it is internally consistent.

Membership / seat

* One wallet receives one seat.
* The first qualifying purchase creates the seat.
* The first purchase does not need to equal the minimum exactly.
* If Genesis minimum is $5 and someone buys $25 first, they become a member and receive one seat.
* If someone buys $5 first, they become a member and can later buy more.
* Additional purchases do not create additional seats.

Membership entry

* Each chapter/era has a minimum entry amount.
* Genesis minimum = $5.
* Later chapters may increase the minimum.
* The minimum is an entry threshold, not a cumulative wallet cap.

Packages

* Packages represent conviction size.
* Packages should be available if under the anti-abuse ceiling.
* Packages should not require waiting for a later chapter if that contradicts the intended model.
* Packages should increase cumulative SYN and cumulative recognition.

Recognition

* Rank derives from cumulative purchases.
* Rank is recognition, not ownership rights.
* Rank should persist across purchases and sale migrations.

Chapter / Era

* Chapter controls minimum entry and SYN rate.
* Chapter may also influence membership-card identity/color/story.
* Chapter should not necessarily block larger purchases.
