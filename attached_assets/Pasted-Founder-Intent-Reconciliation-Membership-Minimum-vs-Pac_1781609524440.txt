Founder Intent Reconciliation — Membership Minimum vs Package Size

Stop before implementing Path A as final doctrine.

The intended model is:

Chapter / Era membership price = minimum entry price.

Package size = how deep someone wants to buy.

They are not the same thing.

Founder intent

Chapter I / Genesis:

* membership entry minimum = $5
* once someone buys, they becomes a member / takes one seat
* they may also buy $25, $100, $500, $2,500, $10,000 packages if they want
* one wallet should not receive multiple seats
* but one wallet should be able to accumulate more SYN / higher recognition

Future chapter example:

* Chapter II membership entry minimum may become $10
* but buyers should still be able to buy larger packages
* Chapter III membership entry minimum may become $15
* same logic

So:

One seat per wallet = identity doctrine.

$5 Genesis minimum = entry doctrine.

$25 / $100 / $500 / $2,500 / $10,000 packages = conviction / recognition / treasury growth.

Do not collapse minimum entry into maximum purchase.

The current deployed behavior appears wrong for this product model because maxUsdcPerAddressPerEra[Genesis] = $5, which makes $5 both the minimum and maximum.

The package UI shown on /join clearly communicates that larger packages are part of the live membership experience, not only a far-future Era II concept.
