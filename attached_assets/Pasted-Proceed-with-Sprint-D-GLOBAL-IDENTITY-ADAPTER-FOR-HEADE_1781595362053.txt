Proceed with Sprint D:

GLOBAL IDENTITY ADAPTER FOR HEADER / MOBILEBAR

Goal

Create a safe global identity adapter and use it only where needed to make the global entry CTAs state-aware.

This sprint should fix the current problem:

Connect ≠ membership.

A user can connect/open context before becoming a member.

A user becomes a Syndicate Member only by buying through Sale V2 and receiving SYN.

Current truths to preserve

Sale V2 is live-but-unaudited / early.
Sale V1 is sealed/history.
CommissionRouter is unset / referral pending.
Archive1155 is artifacts only.
SeatRecord721 is future.
Web2 + Web3 onboarding is future.
Current wallet remains injected/wagmi.
No contract changes.
No route changes.
No sale/mint/referral logic changes.
No full redesign.
No invented data or rewards.

A. Build global identity adapter

Create the safest adapter, likely:

src/lib/use-global-identity.ts

Use existing proven sources only:

wallet/session state
holder index
mounted/SSR-safe gate
existing member detection logic
existing identity loading logic

Do not add new chain reads if the same holder-index query is already shared.

Adapter must expose:

isConnected
isLoadingIdentity
isMember
isConnectedNonMember
address if available
primaryMembershipLabel
dashboardLabel
shouldShowMySyndicate
shouldShowBuyMoreSyn
shouldShowBecomeMember

State rules:

Disconnected visitor:

Connect = wallet action
Join The Syndicate = membership purchase route

Identity loading:

neutral fallback
do not flash Become Member or Buy More SYN incorrectly

Connected non-member:

dashboard may be accessible
membership CTA = Become a Syndicate Member

Member:

dashboard CTA = My Syndicate
membership continuation CTA = Buy More SYN

MobileJoinBar:
