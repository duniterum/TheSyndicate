## OBJECTIVE

Create a canonical Protocol Metrics Registry.

The purpose is:

* one metric
* one definition
* one formula
* one verification path
* many surfaces

No metric should need to be rediscovered later.

## METRIC MODEL

Every metric should be represented as a reusable protocol primitive.

For each metric define:

* id
* label
* category
* type:

  * RAW
  * DERIVED
  * AGGREGATE
* status:

  * LIVE
  * PARTIAL
  * PENDING
* unit
* source
* formula
* verification source
* legal sensitivity
* display priority
* recommended surfaces
* dependencies

## VERIFICATION

Every metric must answer:

1. What is it?
2. Where does it come from?
3. How is it calculated?
4. Can it be verified?
5. What are its dependencies?
6. Can it create legal confusion?
