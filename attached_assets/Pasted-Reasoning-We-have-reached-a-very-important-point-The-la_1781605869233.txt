Reasoning

We have reached a very important point.

The last successful sprints did not add features.

They reduced duplication, centralized logic, clarified identity, clarified navigation, and extracted existing intelligence from the protocol.

That pattern is working.

The biggest risk now is not missing functionality.

The biggest risk is accidentally creating a second journey system when the codebase already contains most of the journey.

Before implementing Journey Assembly V2, we need one final salvage/reconciliation pass.

The goal is to determine:

* what already exists
* what should survive
* what should be upgraded
* what should be retired
* what is truly missing

without inventing a parallel system.

Prompt

Use the current codebase, Full Protocol View, completed sprint history, and current live implementation.

Do not implement.

Do not redesign.

Do not run a broad audit.

Do not create new journey systems.

Goal

Determine whether the existing journey/progression components can be assembled into the final Syndicate journey experience.

Important principle

Prefer:

reuse
unify
simplify

over:

invent
duplicate
replace
