C) PROTOCOL MEMORY / CHRONICLE CLASSIFICATION

Add or formalize the classification model, but do not overbuild.

Important ruling:

Activity records everything.

Chronicle is not a transaction log.

Chronicle is protocol memory.

It has two registers:

1. Protocol / Institutional Register

For meaningful protocol-level events:

* token deployment
* sale deployment
* first liquidity
* first member
* first burn
* first artifact
* first milestone
* first external campaign spend
* first treasury deployment
* major Vault movement
* major Operations spend
* major LP change
* future protocol-owned asset acquisition

2. Member / Living Register

For meaningful member-level events:

* new member
* early member milestone
* member reaches recognition milestone
* member witnesses milestone
* repeat participation
* artifact participation
* future recognition events

Classification rule:

Activity → Signal → Memory → Chronicle → Story

For now, because Signals are not implemented, do not fake Signals.

Instead:

* keep current Chronicle entries as manually curated verified facts.
* add clear type/category fields if needed:

  * protocol
  * member
  * genesis
  * treasury
  * operations
  * liquidity
  * artifact
  * milestone
  * burn
  * founder-action
  * system-wallet-action

Founder/system wallet rule:

Every founder or system wallet action appears in Activity if detected.

But not every action becomes Chronicle.

Founder/system actions become Chronicle only if they are meaningful institutional events.

Examples:

* founder sends routine funds → Activity only.
* founder funds Operations for first campaign → Protocol Memory candidate.
* Operations wallet spends for first campaign → Protocol Chronicle candidate.
* Treasury invests into asset/company/token → Protocol Chronicle candidate.
* Vault movement crossing major threshold → Protocol Chronicle candidate.

Do not glorify founder actions.

Do not make founder wallet the story.

The protocol is the story.

Founder/system actions are institutional facts when they materially affect the protocol.
