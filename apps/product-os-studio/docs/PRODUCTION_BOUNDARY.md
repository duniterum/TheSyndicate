# Production Boundary

## Product OS Studio

Product OS Studio is a visual prototype and product blueprint workspace. It may contain:

- visual prototype work
- product blueprint material
- simulated roles
- mock data
- future modules
- founder review surfaces

It may use localStorage or static data to simulate role states, product flows, dashboards, ledgers, toolkits, and recognition surfaces. Those simulations are not authority.

## Production App

The production app is the real live site. It owns:

- chain truth
- real routes
- real contract reads
- release guards
- real wallet flows
- legal/disclosure boundaries

Production surfaces must be grounded in live contracts, approved configuration, tested data adapters, and release checks.

## Rule

The Product OS Studio can inspire production, but production must only receive approved, tested, truth-grounded slices.

No Studio file should be copied directly into production `src/` as production truth.
