# Replit Sync Guide

Replit 2 should place or sync the standalone Product OS Studio prototype source inside:

```txt
apps/product-os-studio/
```

This folder is intentionally isolated from the production app.

## Rules For Replit 2

- Do not modify production `src/`, `contracts/`, or live route files unless a separate approved task exists.
- Preserve visible labels for simulated work: `SIMULATED`, `FUTURE`, `NOT LIVE`, `PROTOTYPE ONLY`, `BACKEND REQUIRED`.
- Report any assumptions, missing source truth, or fake placeholders.
- Do not add secrets.
- Do not add real private keys.
- Do not add fake live addresses.
- Do not mark simulated data as live.

## Expected Prototype Contents

The folder may later contain standalone visual/product work such as public proof surfaces, member app experiments, Founder Console prototypes, role-gating simulation, recognition boards, Fire / Proof-of-Burn concepts, ledgers, toolkit actions, protocol graphs, press/share/learn screens, and member identity or sigil work.

Those materials remain prototype materials until Codex bridges an approved, tested, truth-grounded slice into production.
